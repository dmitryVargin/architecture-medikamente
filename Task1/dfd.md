# Task 1 — Диаграммы потоков данных (DFD)

> Визуальные DFD-диаграммы для каждого процесса представлены в файле [dfd.drawio](dfd.drawio) (6 страниц — по одной на каждый процесс).

## Процесс 1: Запись пациента на приём

**Чувствительные данные в процессе:**
- ФИО, дата рождения, телефон, e-mail — PII (персональные данные)
- Хронические заболевания (при расширенной форме) — Медицинская тайна

**Текущие меры защиты:** доменная аутентификация (AD). Других нет.

**Проблемы:**
- Данные передаются устно или по нешифрованному e-mail
- Excel-файл доступен всем пользователям домена
- Нет аудита — кто открывал, изменял, копировал файл

---

## Процесс 1а: Заключение клиентского контракта

**Чувствительные данные в процессе:**
- Паспортные данные, СНИЛС — особая категория ПДн (152-ФЗ)
- Контактные данные (телефон, адрес) — PII
- Условия договора (стоимость, перечень услуг) — коммерческая тайна + косвенно медицинская тайна

**Текущие меры защиты:** доменная аутентификация (AD), бумажный экземпляр — в папке.

**Проблемы:**
- Контракт в PDF хранится на общем диске без шифрования
- Нет учёта согласий на обработку ПДн (152-ФЗ обязывает)
- Любой аутентифицированный сотрудник может открыть контракт любого пациента
- Бумажные копии хранятся без контроля доступа
- Нет срока хранения / политики уничтожения

---

## Процесс 2: Приём пациента врачом

**Чувствительные данные в процессе:**
- Диагноз, анамнез, жалобы, результаты анализов — Медицинская тайна (высший уровень)
- ФИО пациента в привязке к диагнозу — особая категория ПДн по 152-ФЗ

**Текущие меры защиты:** нет (файлы на диске без шифрования, нет контроля доступа).

**Проблемы:**
- Медицинская карта в открытом виде на файловом сервере
- Любой аутентифицированный пользователь домена может прочитать файл
- Направление в лабораторию содержит диагноз и передаётся без защиты
- Результаты анализов приходят по e-mail — нет шифрования

---

## Процесс 3: Обработка платежа

**Чувствительные данные в процессе:**
- Данные платёжной карты (PAN, имя держателя) — Финансовые данные (PCI DSS)
- Сумма и услуга — в связке с ФИО пациента раскрывает медицинскую тайну
- Зарплатные данные сотрудников — Персональные данные / коммерческая тайна

**Текущие меры защиты:** ККМ сертифицирована, 1С — базовая парольная защита файлов.

**Проблемы:**
- Двойное ручное ведение (Excel + 1С) — риск расхождений и утечки через Excel
- Передача данных ККМ по внутренней сети без шифрования (OLE/TCP)
- Файловый режим 1С — БД в виде файлов без шифрования на диске
- Нет разделения доступа: бухгалтер видит все платежи всех пациентов

---

## Процесс 4: Управление складом и ТМЦ

**Чувствительные данные в процессе:**
- Поставщики, цены закупок — коммерческая тайна
- В случае учёта медикаментов — сведения о номенклатуре, косвенно раскрывающие специализацию клиники

**Текущие меры защиты:** базовая доменная аутентификация.

**Проблемы:**
- Дублирование учёта (Excel + 1С)
- Нет версионности документов

---

## Процесс 5: HR и кадровый учёт

**Чувствительные данные в процессе:**
- Паспортные данные, СНИЛС, ИНН, адрес, зарплата — особая категория ПДн
- Трудовые договоры с условиями оплаты — конфиденциально

**Текущие меры защиты:** 1С парольная защита, бумажные документы в сейфе.

**Проблемы:**
- Отсутствие шифрования в 1С-файлах на диске
- Доступ к кадровой 1С не ограничен ролями внутри системы
- Электронная переписка с банком по незащищённому каналу
