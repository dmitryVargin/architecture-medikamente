## 3. Инструменты защиты данных

### 3.1 Шифрование at-rest

**PostgreSQL — шифрование на уровне колонок (pgcrypto)**
```sql
-- Шифрование при записи
INSERT INTO medical_records (patient_uuid, diagnosis)
VALUES ($1, pgp_sym_encrypt($2, current_setting('app.encryption_key')));

-- Расшифровка при чтении (только авторизованный сервис)
SELECT pgp_sym_decrypt(diagnosis::bytea, current_setting('app.encryption_key'))
FROM medical_records WHERE patient_uuid = $1;
```
- Применяется для колонок: diagnosis, anamnesis, test_results, prescription
- Ключ шифрования хранится в HashiCorp Vault, не в конфиге приложения

**HashiCorp Vault — управление ключами**
- Хранение всех ключей шифрования
- Transit Secrets Engine: шифрование/расшифровка через API (данные не покидают Vault)
- Автоматическая ротация ключей каждые 90 дней
- Аудит всех обращений к ключам

**LUKS (Linux Unified Key Setup) — шифрование дисков**
- Шифрование всех блочных устройств в Kubernetes-нодах и на сервере БД
- Алгоритм: AES-256-XTS
- Применяется как дополнительный уровень защиты

**ClickHouse — шифрование колонок (аналитическое хранилище)**
```sql
CREATE TABLE medical_analytics (
    patient_uuid UUID,
    diagnosis String CODEC(AES_128_GCM_SIV)
) ENGINE = MergeTree();
```

**MinIO (объектное хранилище) — шифрование объектов**
- SSE-S3: серверное шифрование AES-256
- SSE-C: ключ предоставляет клиент (для Landing Zone с критическими данными)

### 3.2 Шифрование in-transit

**TLS 1.3 — все HTTP-соединения**
- Обязателен для всех клиент-сервер соединений
- TLS 1.0 и 1.1 отключены
- HSTS (HTTP Strict Transport Security) включён
- Сертификаты: внутренний CA (cert-manager в Kubernetes) + Let's Encrypt для публичных endpoints

**mTLS (Mutual TLS) — межсервисное взаимодействие**
- Все вызовы между микросервисами через mTLS
- Реализация: Istio Service Mesh
- Автоматическая ротация сертификатов каждые 24 часа

**Шифрование в Kafka**
- TLS для транспортного уровня
- Шифрование payload для топиков с MEDICAL_SECRET: AES-256-GCM на уровне приложения

**VPN / PrivateLink для Lab API**
- WireGuard VPN для соединения с лабораторией анализов
- Или: HTTPS mTLS с certificate pinning

### 3.3 Токенизация и псевдонимизация

**Токенизация платёжных данных**
- PAN (номер карты) никогда не хранится в системе
- При оплате: карта токенизируется на стороне Payment Gateway
- В системе хранится только token, привязанный к patient_uuid

**Псевдонимизация ПДн для аналитики**
- При загрузке в Bronze Layer: ФИО → patient_uuid (UUID v4)
- Маппинг UUID → реальные ПДн хранится только в оперативной БД
- Обратное раскрытие невозможно без доступа к оперативной БД

### 3.4 Маскирование данных

**Маскирование в логах**
- Фильтры в Logback/Log4j: телефон, email, ФИО заменяются маской
- Пример: `+7 900 123-45-67` → `+7 9** ***-**-67`
- Реализация: custom log filter + Apache Log4j Rewrite Appender

**Маскирование в API-ответах**
- API Gateway фильтрует чувствительные поля в ответах для непривилегированных ролей
- Пример: ресепшен видит первые 2 буквы диагноза, полный текст — только врач
